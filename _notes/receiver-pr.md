## P型受信總機

- **P = Proprietary**（共通信號式）
- 每一回路代表一個**火警分區**
- 探測器動作 → 回路短路 → 總機只知道「哪個分區」有火警
- 無法判斷是哪一個探測器動作
- 回路數 = 火警分區數

### P型回路配線
- 採用**2線式**配線
- 一般配線（非耐燃耐熱可）
- 線路斷線 → **該回路失效**

## R型受信總機

- **R = Record**（記錄式/定址式）
- 每個探測器有**獨立地址編碼**
- 探測器動作 → 總機精確顯示**哪一個探測器**動作
- 可監控每個探測器的狀態

### R型回路配線
- 採用**迴路式（環形）**配線
- 單點斷線 → 仍可從另一端通訊（**不失效**）
- 但兩點斷線 → 中間探測器失效

## P型 vs R型 比較

| 比較項目 | P型 | R型 |
|---------|-----|-----|
| 地址功能 | 無（只知分區）| **有**（知道哪個探測器）|
| 配線方式 | 2線式 | 迴路式（環形）|
| 斷線影響 | 該回路失效 | 單點斷線仍可通訊 |
| 成本 | 較低 | 較高 |
| 適用規模 | 中小型建築 | 大型/複雜建築 |
| 火警定位 | 只到分區 | 精確到探測器 |
| 擴充性 | 差（要增加回路）| 好（增加地址即可）|

## 受信總機設置規定

| 項目 | 規定 |
|------|------|
| 設置位置 | **防災中心**或守衛室（值班室）|
| 操作面 | 距牆 **80cm 以上** |
| 高度 | 操作開關距地面 **0.8m-1.5m** |
| 備用電源 | 監視狀態 **10小時** + 警報狀態 **10分鐘** |

## 記憶口訣

> **「P型看分區，R型看地址」**
> P = 便宜但粗略，R = 貴但精準
